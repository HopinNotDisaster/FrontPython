Django 1.8.5 版本发行说明¶

2015 年 10 月 3 日

Django 1.8.5 修复了 1.8.4 中的几个错误。

漏洞修复¶

  • 改进了开发服务器的自动重新加载功能，使其更加健壮（#24704）。
  • 修复了包含既是外键又是主键的字段的模型中一些删除查询中的 AssertionError （#24951）。
  • 修复了一些复杂查询中的 AssertionError （#24525）。
  • 修复了使用 GenericForeignKey 时的迁移崩溃问题（#25040）。
  • 在初始情况下未激活翻译时，使 translation.override() 清除被覆盖的语言（#25295）。
  • 修复了在使用 ModelAdmin.list_display 中的值与模型上的反向字段冲突时导致崩溃的问题（#25299）。
  • 修复了非 argparse 管理命令选项的自动补全问题（#25372）。
  • 在新创建的迁移中，按字母顺序排列了 from django.db import migrations, models 语句中的导入项（#25384）。
  • 修复了在 MySQL 上添加具有不可哈希默认值的文本或 blob 字段时导致迁移崩溃的问题（#25393）。
  • 将 Count 查询更改为执行 COUNT(*)，而不是像 Django 1.8 之前的版本一样执行 COUNT('*') （#25377）。这可能会在某些数据库上修复性能回退问题。
  • 修复了在使用 values() 和 values_list() 进行自定义查询集链的问题（#20625）。
  • 将对反向关系的 未保存的模型实例分配数据丢失检查 移动到了 Model.save() 中（#25160）。
  • 在验证模型表单集时，重新将内联外键添加到表单实例中（#25431）。
  • 允许在使用 set_autocommit(False) 禁用自动提交后使用 ORM 写入方法（#24921）。
  • 修复了 Oracle 上的 manage.py test --keepdb 选项（#25421）。
  • 修复了在模型上具有相同“to”模型并且 related_name 设置为 '+' 时的多个多对多字段的不正确查询（#24505，#25486）。
  • 修复了对包装模型的 SimpleLazyObject 进行序列化的问题（#25389）。
